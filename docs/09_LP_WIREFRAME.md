# LP Wireframe

## 1. Purpose

このワイヤーフレームは、最終ビジュアルを作る前に、情報の順番、余白、導線、事例の見せ方を確認するための構造設計である。

対象は、名刺のQRコードからスマートフォンでアクセスする地元の中小企業・個人事業主を中心とする。

そのため、PC版を縮小するのではなく、スマートフォン版を正本として設計する。

## 2. Experience Goal

閲覧者が30〜60秒で次を理解できること。

1. AIに詳しいだけの人ではなく、現場の面倒を改善する人
2. Excel、書類、検索、SNSなど身近な作業を相談できる
3. 実際に仕組みを作っている
4. 技術者であり音楽家でもある、記憶に残る人物
5. まだ課題が整理できていなくても相談できる

## 3. Page Rhythm

大規模ホールのような広大さではなく、小規模ライブスペースの距離感を作る。

- 一画面へ情報を詰め込みすぎない
- セクション間に静かな暗転を入れる
- 長大な余白ではなく、会話が途切れない程度の間を作る
- 重要なコピーは中央揃えにしすぎず、話しかけるように左揃えにする
- カードを均一に大量配置せず、代表事例は大小をつけて見せる

## 4. Navigation

### Desktop

左：Signal Fロゴ＋`SYSTEM / SOUND`

右：

- Solutions
- Works
- About
- Contact

Contactのみ、細い赤線または小さな赤点を付ける。

### Mobile

- 左：ロゴ
- 右：Menu
- 画面下部に小さな固定CTA `相談する`

固定CTAは常時目立たせず、ファーストビューを一定量スクロールした後に表示する想定。

## 5. Section Structure

## 5.1 Hero — Small Stage

### Goal

肩書きより先に、何をしてくれる人かを伝える。

### Content

Eyebrow：

`AI SOLUTION ENGINEER / CONTRABASS PLAYER`

Main copy：

**仕事の中にある「毎回面倒」を、  
AIと小さな仕組みで改善します。**

Sub copy：

Excel、情報検索、書類作成、SNS運用、小さな業務アプリ。現在のやり方を聞き、現場で無理なく使える形まで設計します。

CTA：

- Primary：`できることを見る`
- Secondary：`相談する`

Visual：

- 黒い背景
- 画面右または奥に木の曲面を感じる光
- Signal Fが大きく置かれる
- 信号線が次セクション方向へ続く
- 赤い点は一つだけ

### Mobile order

1. Eyebrow
2. Main copy
3. Sub copy
4. CTA
5. Signal F visual

ロゴビジュアルを先に見せず、言葉を先に読む構造とする。

---

## 5.2 Positioning — AI Is a Tool

### Heading

**AIを入れることが、目的ではありません。**

### Body

現在の作業を見て、AI、Excel、GAS、Webアプリ、APIなどから必要な方法を選ぶ。

### Visual structure

横一列または縦並びで、日常作業を示す短い言葉を置く。

`探す → 転記する → 並べる → 作る → 確認する → 投稿する`

その中の一部に細い赤い下線を置き、改善対象を見つける行為を表す。

---

## 5.3 Solutions — What Can Be Discussed

6枚の均一カードではなく、2列×3段を基本としながら、スマホでは縦並びにする。

### Items

1. 情報を探しやすくする
2. 書類作成を楽にする
3. 定型作業を自動化する
4. SNS運用を仕組み化する
5. 小さな業務アプリを作る
6. AI活用の相談に乗る

### Card anatomy

- 小さな番号
- 相談内容
- 日常語での一文
- 技術名は原則表示しない

カードの背景は黒に近いチャコール。ホバー時に光らせず、木目色の細い枠または上辺だけを表示する。

---

## 5.4 Featured Works — Before / After Stage

### Heading

**作ったものではなく、変えた作業。**

### Layout

代表事例は最大6件。

すべて同じサイズのカードへ入れず、次のリズムを使う。

- 1件目：幅いっぱいの主役
- 2・3件目：左右2カラム
- 4件目：幅いっぱい
- 5・6件目：左右2カラム

スマホはすべて縦並び。

### Case card anatomy

1. Status：`OPERATING` / `MVP` / `DEMO` / `PRIVATE CASE`
2. Display title
3. One-line result
4. Before / After mini diagram
5. Main image or video
6. Current state and limitations
7. `事例を見る`

## 5.5 Process — Close Collaboration

### Heading

**小さく聞いて、小さく試して、一緒に整える。**

### Steps

1. 聞く
2. 整理する
3. 試作する
4. 使って確認する
5. 改善する

### Visual

一般的な横長タイムラインではなく、一本の弦または信号線がゆるく上下しながら各ステップをつなぐ。

赤い点は現在地ではなく、課題を発見するStep 1付近だけに使用する。

---

## 5.6 Capability — Tools Behind the Work

### Heading

**方法は、課題に合わせて選びます。**

目的別の4グループで掲載する。

- AI活用
- 自動化・連携
- アプリ開発
- 運用設計

技術名はタグとして小さく表示する。ロゴ一覧のように派手に並べない。

---

## 5.7 About — SYSTEM / SOUND

### Layout

Desktop：左右2カラム

- 左：AIソリューションエンジニアとしての写真または作業風景
- 右：コントラバス演奏写真

中央または境界にスラッシュ `/` とSignal Fを置く。

Mobile：

1. プロフィール本文
2. 技術側写真
3. 音楽側写真
4. SYSTEM / SOUNDの短いストーリー

### Copy direction

エンジニアと音楽家を別の経歴として並べず、複雑なものを整理して心地よく動く状態を作るという共通点でつなぐ。

---

## 5.8 Contact — One Light On

### Heading

**その作業、少し楽にできるかもしれません。**

### Body

依頼内容が固まっていなくてもよいことを伝える。

例：

- 毎回この入力が面倒
- Excelが増えすぎて探せない
- AIで何ができるかだけ聞きたい
- SNS投稿を続けられる形にしたい

### CTA

Primary：`相談内容を送る`

Links：

- note
- GitHub
- Threads
- Email / Form

### Visual

画面の大部分は黒。左上または中央に柔らかなスポットライト。Signal Fの赤い一点だけが残る。

## 8. Responsive Rules

- Breakpointの数を増やしすぎない
- 主要基準：Mobile `< 768px`、Desktop `>= 768px`
- 本文幅は最大65〜72文字程度
- Hero見出しはスマホで3〜5行を許容する
- 事例カード内のBefore／After図はスマホで縦並びにする
- 固定CTAがフォームやフッターを隠さないよう、Contact付近で非表示にする
- 動画は自動再生せず、短いループを使う場合も `prefers-reduced-motion` に対応する

## 9. Accessibility

- 黒背景上の本文コントラストを確保する
- 木目色を本文色に使用しない
- 赤だけで状態を示さない
- SVGロゴへ代替テキストを設定する
- キーボード操作で全リンクへ到達できる
- フォーカスリングを消さない
- 動きは停止可能、またはOS設定に従う
