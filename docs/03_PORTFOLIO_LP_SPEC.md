# Portfolio LP Specification

## 1. Goal

名刺を受け取った地元の中小企業・個人事業主がスマートフォンでアクセスし、短時間で次を理解できるLPを作る。

1. どのような人物か
2. 何を相談できるか
3. 実際に何を作ったか
4. 自分の業務にも使えそうか
5. どう連絡すればよいか

作品を大量に並べるのではなく、相談への心理的な距離を縮めることを優先する。

## 2. Experience Concept

サイトを訪れた人が、小規模なライブスペースへ入り、目の前で演奏と説明を受けているような体験を目指す。

- ファーストビュー：暗い部屋へ入る
- サービス説明：スポットライトが課題へ当たる
- 実績：一つずつ演奏曲を紹介するように見せる
- プロフィール：演奏者／制作者本人が見える
- CTA：終演後に直接声をかけられる距離感

## 3. Site Structure

### 3.1 Header

表示項目：

- SYSTEM / SOUND ロゴ
- Solutions
- Works
- About
- Contact

スマートフォンでは、常時大きなメニューを置かない。小さなメニューと固定の相談ボタンを使用する。

### 3.2 Hero

#### Main Copy

**仕事の中にある「毎回面倒」を、  
AIと小さな仕組みで改善します。**

#### Supporting Copy

Excel、情報検索、書類作成、SNS運用、業務アプリなど。現在の作業を聞きながら、現場に合う方法を一緒に考え、試作します。

#### Labels

- AI SOLUTION ENGINEER
- SYSTEM / SOUND

#### CTA

Primary：`できることを見る`

Secondary：`相談する`

#### Visual

- Stage Blackの背景
- スポットライトに照らされたF字孔の抽象ライン
- ラインの一部は周波数波形や信号線へ変化
- 背景の一部に、艶のある木部の曲面
- 赤い小点を一か所だけ配置

人物や楽器全体の写真を最初から大きく出す案と、抽象ビジュアルだけで始める案を比較テストする。

### 3.3 Problem Statement

#### Heading

**AIを導入することが目的ではありません。**

#### Content

仕事の中にある次の繰り返しを見つける。

- 探す
- 転記する
- 入力する
- 並べる
- 集計する
- 確認する
- 投稿する

AIが必要ならAIを使い、Excelで十分ならExcelを使う。現在の様式や運用を確認し、効果の高い部分から小さく改善する。

### 3.4 Solutions

6枚のカードまたは縦スクロール型の紹介とする。

#### 情報を探しやすくする

複数のExcel、資料、社内メモへ散らばった情報を、横断して探せるようにする。

#### 書類作成を楽にする

Excel、Word、PowerPoint、PDFなど、現在の様式と元データを確認し、入力や生成を効率化する。

#### 定型作業を自動化する

転記、集計、保存、通知、ファイル整理などの繰り返しを仕組み化する。

#### SNS運用を仕組み化する

投稿作成、予約、画像管理、実行、履歴確認、分析を一つの流れにする。

#### 小さな業務アプリを作る

市販サービスでは合わない作業を、ブラウザで使える簡易ツールとして試作する。

#### AI活用を一緒に考える

AI向きの作業と、AIを使わない方がよい作業を切り分ける。

### 3.5 Featured Works

初期公開では4〜6件に限定する。

#### Candidate A：ラック図面作成支援Webアプリ

- Before：Excel図形の配置と装置名・ポートの手入力
- Solution：ドラッグ操作で装置を配置できるブラウザアプリ
- Value：手作業の図面作成を、現場向けUIへ置き換えた
- Notes：機密情報はダミーデータへ置換

#### Candidate B：社内情報横断検索ツール

- Before：複数人が追記したExcelの虎の巻を個別に検索
- Solution：複数情報をまとめて検索できる仕組み
- Value：探す作業そのものを改善
- Notes：勤務先情報や実データは公開しない

#### Candidate C：Threads自動投稿・運用支援

- Before：投稿準備、予約、実行、履歴確認を分散管理
- Solution：ChatGPT、GitHub、GitHub Actions、Threads APIを連携
- Value：会話から投稿運用へつなげる一連の仕組み

#### Candidate D：IrodoriTTS Studio

- Before：長文音声の分割、生成、保存、変換、結合が複雑
- Solution：制作工程をまとめたUIと管理機能
- Value：AIモデル単体ではなく、実用工程を設計

#### Candidate E：画像整理・加工ツール

- スクリーンショット整理
- 背景除去
- 画像分割
- ファイル管理

#### Candidate F：AIクリエイティブ

- 音楽・音声制作
- AI漫画
- ゲーム
- キャラクター制作

補助事例として扱い、業務改善事例より前へ出しすぎない。

### 3.6 Work Card Format

各事例を次の順番で表示する。

1. 困っていたこと
2. 以前の方法
3. 作ったもの
4. 改善された流れ
5. 使用技術
6. 制約・注意点
7. 今後の改善
8. note／GitHub／デモ

効果数値は実測できる場合のみ掲載する。

### 3.7 Process

#### 1. 小さく聞く

現在の作業、ファイル、入力元、変更箇所、頻度を確認する。

#### 2. 改善箇所を整理する

全自動化ではなく、効果とリスクのバランスがよい部分を選ぶ。

#### 3. 小さく試作する

限定されたデータで、まず動くものを作る。

#### 4. 実際に使う

操作性、誤入力、例外、現場との相性を確認する。

#### 5. 改善して定着させる

必要に応じてログ、マニュアル、運用ルールを整える。

### 3.8 Technology

技術名は目的別に整理し、主役にしない。

- AI：ChatGPT、生成AI、文書処理、音声、画像、RAG、エージェント
- Automation：API、GAS、GitHub Actions、Python、MCP
- App：HTML、CSS、JavaScript、Webアプリ、UI設計、Unity
- Operation：GitHub、ログ、テスト、エラー対応、マニュアル

### 3.9 Profile

#### Heading

**Engineer by structure. Musician by instinct.**

#### Draft

AIを使うこと自体ではなく、実際に人が使い続けられる状態まで設計することを重視しています。

### 3.11 Final CTA

#### Heading

**その作業、少し楽にできるかもしれません。**

#### Copy

依頼内容が決まっていなくても問題ありません。

「毎回この作業が面倒」  
「AIで何ができるか知りたい」  
「Excelや資料が増えすぎている」

という段階から相談できます。

#### CTA

`相談内容を送る`

補助リンク：Threads、note、GitHub、メールまたはフォーム。

## 4. Interaction and Motion

- 初回表示でF字孔の線がゆっくり描かれる
- スクロールに伴い、直線的な信号線が木の曲線へ変わる
- 実績カードへスポットライトが移るように明度を変える
- 赤はCTA、現在位置、録音ランプ的な一点だけに使う
- 過剰なパララックスは使わない
- `prefers-reduced-motion` に対応する

## 5. Responsive Requirements

名刺のQRコードからの流入を想定し、スマートフォンを先に設計する。

- 360px幅から破綻しない
- 本文16px以上を基本とする
- CTAのタップ領域を44px以上にする
- 画像をWebP／AVIFで最適化する
- ヒーロー表示を3秒以内に理解できる情報量へ抑える
- GitHubやnoteへ移動しなくても、概要がサイト内で分かるようにする

## 6. Accessibility

- 背景と文字のコントラストを確保
- 木目上へ直接長文を置かない
- キーボード操作に対応
- フォーカス状態を赤だけで表現しない
- altテキストを設定
- 動画や音声は自動再生しない

## 7. SEO and Metadata

- title：AIソリューションエンジニア goro | SYSTEM / SOUND
- description：業務改善、AI導入、自動化、小規模Webアプリの相談と、コントラバス／ベースを中心とした音楽活動を紹介。
- OGP：F字孔、黒、木、赤を使用した専用画像
- favicon：単純化したF字孔シンボル
- JSON-LD：Person、ProfessionalServiceを検討

## 8. Success Criteria

初期公開時に次を満たす。

- 相談できる内容がスマートフォンで1分以内に分かる
- 代表事例が4件以上ある
- GitHubのコードを読まなくても価値が伝わる
- AIと音楽の二面性が不自然に分断されていない
- 問い合わせ方法が明確
- 名刺とサイトの世界観が一致している
